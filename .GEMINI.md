# CLAUDE.md — DCLG Toy Experiment Implementation Guide

## 프로젝트 목표

IP-Adapter 기반 단일 이미지 생성에서, 두 객체(기사/오크)의 Cross-Attention Map이
공간적으로 겹치는 현상(키메라)을 억제하는 Guidance 기법을 구현하고 검증한다.

**Fail-Fast 원칙:** 단일 이미지 5장 실험에서 효과가 없으면 즉시 폐기한다.

## 구현 명세

### 1. hook_manager.py

IP-Adapter의 Cross-Attention 레이어에서 Attention Weight를 추출하는 Forward Hook을 관리한다.

**해야 할 것:**
- `unet`의 named modules를 순회하여 `attn2` (cross-attention) 레이어를 찾는다
- `register_forward_hook`으로 각 레이어의 attention weight를 캡처한다
- 캡처 대상: Up-block의 cross-attention 레이어만 (down/mid 제외)
  - 모듈 이름 필터: `"up_blocks"` 포함 AND `"attn2"` 포함
- IP-Adapter는 attention weight를 두 번 계산한다 (text용, image용)
  - image용 weight만 추출해야 한다 (두 번째 호출)
  - 호출 카운터로 구분하거나, key dimension으로 구분한다

**저장 형식:**
```python
# hook_manager가 채워주는 딕셔너리
captured = {
    "layer_name": {
        "attn_A": Tensor[B, heads, HW, seq_A],  # 기사 ref에 대한 weight
        "attn_B": Tensor[B, heads, HW, seq_B],  # 오크 ref에 대한 weight
    }
}
```

**주의:**
- Hook은 매 forward마다 초기화해야 한다 (이전 스텝 값 누적 방지)
- `detach()` 없이 저장해야 gradient가 흐른다

### 2. dclg_pipeline.py

메인 파이프라인. diffusers의 `StableDiffusionPipeline`을 상속하거나 래핑한다.

#### 2-1. 레이어 선택 전략

Up-block에서 레이어 해상도를 기준으로 필터링한다.

```python
TARGET_SPATIAL_SIZE = 16  # 이 값 이상의 HW를 가진 레이어만 사용
# (HW = H*W, H=W=16이면 HW=256)
```

레이어별 실제 해상도는 Hook에서 attn weight shape의 dim=2(HW)로 확인한다.

#### 2-2. Chimera Loss 계산

```python
def compute_chimera_loss(captured: dict) -> torch.Tensor:
    """
    각 레이어에서 attn_A와 attn_B의 공간적 overlap을 계산한다.
    
    Steps:
    1. 각 레이어의 attn_A를 heads 차원 평균 → [B, HW, seq_A]
    2. seq 차원 평균 또는 max → [B, HW]  (공간 맵)
    3. 동일하게 attn_B → [B, HW]
    4. Hadamard product → [B, HW]
    5. 공간 합산 → scalar
    6. 모든 레이어의 loss 합산
    """
```

**주의:**
- Softmax는 이미 attention 계산 시 적용되어 있으므로 추가 적용 금지
- seq 차원 집계: 먼저 `mean`으로 시작, 효과 없으면 `max`로 교체
- Loss에 `log`를 씌우는 변형도 실험 (overlap이 작을 때 gradient 강화)

#### 2-3. Latent Update

```python
def apply_dclg_guidance(x_t, loss, lambda_t):
    """
    x_t: requires_grad=True 상태여야 함
    loss: compute_chimera_loss의 출력
    lambda_t: 현재 스텝의 guidance scale
    """
    grad = torch.autograd.grad(loss, x_t, retain_graph=False)[0]
    x_t_updated = x_t - lambda_t * grad
    return x_t_updated.detach()
```

**주의:**
- `retain_graph=False`가 기본. loss 재사용 없음
- `.detach()` 후 다음 denoising step에 전달
- grad norm이 폭발하면 `torch.nn.utils.clip_grad_norm_` 적용

#### 2-4. Denoising Loop 수정

기존 diffusers 루프에서 매 스텝 다음 순서를 따른다:

```
[스텝 t]
1. x_t.requires_grad_(True)
2. Hook 초기화
3. unet forward (text + image A + image B 동시 처리)
4. Hook에서 attn_A, attn_B 추출
5. lambda_t 계산 (스케줄 참고)
6. lambda_t > 0이면:
   a. compute_chimera_loss()
   b. apply_dclg_guidance()
   c. x_t ← x_t_updated
7. scheduler.step(noise_pred, t, x_t) → x_{t-1}
8. 다음 스텝으로
```

**IP-Adapter 다중 참조 처리:**
- IP-Adapter 공식 구현은 단일 참조 이미지를 가정한다
- 두 이미지를 동시에 주입하려면 두 참조를 배치 차원으로 처리하거나
  첫 번째 forward에서 A만, 두 번째 forward에서 B만 hook하는 방식을 쓴다
- **권장:** 두 번 forward (A, B 분리) → 각각 attn map 저장 → loss 계산
  (연산량 2배지만 구현이 명확함)

### 3. configs/default.yaml

```yaml
model:
  base: "runwayml/stable-diffusion-v1-5"
  ip_adapter: "h94/IP-Adapter"
  ip_adapter_weight: "ip-adapter_sd15.bin"

prompt: "A knight and an orc wrestling fiercely, dynamic pose, cinematic"
negative_prompt: "blurry, deformed, extra limbs"

generation:
  num_inference_steps: 30
  guidance_scale: 7.5
  image_size: 512
  seed: 42

dclg:
  enabled: true
  alpha: 0.5              # 최대 lambda 값
  tau_threshold: 20       # 이 스텝 이후로는 guidance 끔 (30 step 기준)
  target_min_hw: 256      # 대상 레이어 최소 HW (16x16)
  grad_clip: 1.0          # gradient clipping 값, null이면 미적용
  seq_agg: "mean"         # seq 차원 집계 방식: "mean" or "max"

experiment:
  num_images: 5
  lambda_sweep: [0.1, 0.3, 0.5, 1.0, 2.0]  # 이미지별 lambda 값
  save_attn_maps: true
```

## 판정 기준 (Fail-Fast Criteria)

실험 후 다음 기준으로 즉시 판정한다.

| 현상 | 판정 |
|------|------|
| overlap 맵이 실제로 줄어들고, 두 객체가 섞이지 않음 | ✅ 계속 진행 |
| 이미지는 괜찮은데 overlap 감소 효과 없음 | ⚠️ loss 설계 재검토 |
| 객체들이 서로 멀리 떨어지려는 투명 벽 현상 | ⚠️ lambda 줄이기 |
| 이미지 전체에 고주파 노이즈/artifact 발생 | ❌ grad_clip 또는 tau 조정 |
| lambda 어떻게 해도 artifact 해결 안 됨 | ❌ 아이디어 폐기 |

## 구현 시 절대 하지 말 것

- Hook 등록 후 `.detach()`로 저장하지 말 것 (gradient 끊김)
- 모든 레이어에 loss 계산하지 말 것 (down/mid block은 제외)
- lambda를 처음부터 크게 설정하지 말 것 (0.1부터 시작)
- IP-Adapter 가중치를 fine-tuning하지 말 것 (training-free 원칙)
- x_t update 후 scheduler.step에 update된 x_t를 넣는 것을 빠뜨리지 말 것
